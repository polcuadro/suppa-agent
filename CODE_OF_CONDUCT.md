# Code of Conduct

## The short version

Be kind. Be patient. Assume good intent. Don't be a jerk.

## A bit longer

We want suppa-agent to be a place where anyone — regardless of experience, background, language, or identity — can ask for help, report a bug, contribute a fix, or share what they built.

### Things we expect

- Treat others as you'd want to be treated.
- Welcome newcomers and beginners. Everyone was new at some point.
- Disagreement is fine. Personal attacks are not.
- If something isn't clear, ask. If someone asks something that seems obvious, explain anyway.
- Use the issue tracker for bugs and ideas, not for fights.

### Things we don't tolerate

- Harassment, insults, or derogatory comments.
- Discriminatory language or behavior based on identity (gender, race, sexual orientation, disability, religion, nationality, age, etc.).
- Personal attacks, doxxing, sustained disruption.
- Sexual or violent content.
- Spam, off-topic political content, or commercial promotion unrelated to the project.

### What happens if rules are broken

Maintainers may, depending on severity:
1. Edit the offending content.
2. Issue a warning via private message or comment.
3. Lock the issue/PR/discussion.
4. Block the user from the repository.

There's no public shaming process. Most situations resolve with a friendly nudge.

### Reporting

If you experience or witness behavior that violates this Code of Conduct, contact the repo owner privately via the email on their GitHub profile. Reports are handled discreetly. We do not require you to confront the offender directly.

---

This Code of Conduct is loosely inspired by the [Contributor Covenant v2.1](https://www.contributor-covenant.org/version/2/1/code_of_conduct/), simplified for a small project.
